# Access Control Mechanisms

## Introduction

**Access Control** is the process of restricting access to information, systems, and resources so that only authorized users can use them. It is an important part of Information Security because it protects data from unauthorized access, modification, or misuse.

**Access Control Mechanisms** are the methods and techniques used to enforce access control policies.

---

## Definition

**Access Control Mechanisms** are security measures that verify the identity of users and determine what resources they are allowed to access and what actions they can perform.

---

## Types of Access Control Mechanisms

### 1. Identification

Identification is the process by which a user claims an identity.

**Examples:**

* Username
* Employee ID
* Email ID

**Example:** Entering a username while logging into a system.

---

### 2. Authentication

Authentication verifies whether the claimed identity is genuine.

**Methods:**

* Passwords
* PINs
* Smart Cards
* Biometrics (Fingerprint, Face Recognition)

**Example:** Entering a password after providing a username.

---

### 3. Authorization

Authorization determines what an authenticated user is allowed to do.

**Examples:**

* Read files
* Write data
* Delete records
* Access applications

**Example:** A student can view marks but cannot modify them.

---

### 4. Accountability

Accountability ensures that users are responsible for their actions.

**Methods:**

* Audit logs
* Activity monitoring
* System logs

**Example:** Recording who accessed a database and when.

---

## Common Access Control Models

### 1. Discretionary Access Control (DAC)

* Resource owner decides who can access the resource.
* Permissions can be granted or revoked by the owner.

**Example:** A file owner sharing a document with specific users.

---

### 2. Mandatory Access Control (MAC)

* Access is controlled by the system according to predefined security policies.
* Users cannot change permissions.

**Example:** Military systems with classified information.

---

### 3. Role-Based Access Control (RBAC)

* Access permissions are assigned according to user roles.

**Examples:**

* Manager
* Employee
* Administrator

**Example:** HR staff can access employee records, while other employees cannot.

---

### 4. Attribute-Based Access Control (ABAC)

* Access decisions are based on attributes such as:

  * User role
  * Location
  * Time
  * Device type

**Example:** Allowing access only during office hours.

---

## Access Control Process

```text
User
  ↓
Identification
  ↓
Authentication
  ↓
Authorization
  ↓
Access Granted/Denied
  ↓
Accountability (Logging & Auditing)
```

---

## Benefits of Access Control Mechanisms

1. Protects confidential information.
2. Prevents unauthorized access.
3. Maintains data integrity.
4. Supports accountability through auditing.
5. Reduces security risks.
6. Helps achieve Confidentiality, Integrity, and Availability (CIA).

---

## Conclusion

**Access Control Mechanisms** are essential security measures used to regulate access to information systems. They include **Identification, Authentication, Authorization, and Accountability**, along with models such as **DAC, MAC, RBAC, and ABAC**. These mechanisms help ensure that only authorized users can access and use organizational resources securely.
