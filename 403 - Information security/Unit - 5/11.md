# Authentication and Authorization

## Introduction

Authentication and Authorization are two important security processes used to protect information systems. They ensure that only legitimate users can access resources and perform permitted actions.

## 1. Authentication

### Definition

**Authentication** is the process of verifying the identity of a user, device, or system before granting access.

It answers the question:

**"Who are you?"**

### Methods of Authentication

#### 1. Password-Based Authentication

* Uses username and password.
* Most common authentication method.

#### 2. Smart Cards

* Uses electronic cards containing user credentials.

#### 3. Biometric Authentication

* Verifies identity using physical characteristics.

**Examples:**

* Fingerprint
* Iris Scan
* Face Recognition

#### 4. One-Time Password (OTP)

* Generates a temporary password for each login session.

#### 5. Multi-Factor Authentication (MFA)

* Uses two or more authentication methods.
* Provides stronger security.

**Example:** Password + OTP.

### Example

When a user enters a username and password to log into a system, the system verifies the credentials. This process is called **Authentication**.

---

## 2. Authorization

### Definition

**Authorization** is the process of determining what an authenticated user is allowed to access or perform.

It answers the question:

**"What are you allowed to do?"**

### Functions of Authorization

* Grants access rights and permissions.
* Restricts access to sensitive resources.
* Controls user actions according to roles.

### Examples

* A student can view results but cannot modify them.
* An administrator can create, update, and delete records.
* A manager can access departmental reports.

---

## Difference Between Authentication and Authorization

| Authentication                   | Authorization                |
| -------------------------------- | ---------------------------- |
| Verifies user identity           | Determines user permissions  |
| Answers "Who are you?"           | Answers "What can you do?"   |
| Occurs first                     | Occurs after authentication  |
| Uses passwords, OTPs, biometrics | Uses roles and access rights |
| Confirms identity                | Grants or denies access      |

---

## Importance of Authentication and Authorization

1. Prevent unauthorized access.
2. Protect sensitive information.
3. Ensure data confidentiality and integrity.
4. Enforce security policies.
5. Improve overall system security.

## Conclusion

**Authentication** verifies the identity of a user, while **Authorization** determines the resources and actions that the user is permitted to access. Together, they form the foundation of access control and play a vital role in maintaining information security.
