# Access Control Matrix

## Introduction

An **Access Control Matrix (ACM)** is a security model used to define and manage the access rights of users to various system resources. It specifies **who (subject)** can access **what (object)** and what operations they are allowed to perform.

It is an important mechanism for implementing **access control** in computer systems and databases.

---

## Basic Components of Access Control Matrix

### 1. Subject

A subject is an active entity that requests access to resources.

**Examples:**

* Users
* Processes
* Programs

### 2. Object

An object is a resource that needs protection.

**Examples:**

* Files
* Databases
* Printers
* Applications

### 3. Access Rights

These specify the operations a subject can perform on an object.

**Examples:**

* Read (R)
* Write (W)
* Execute (X)
* Delete (D)

---

## Structure of Access Control Matrix

The rows represent **subjects (users)** and the columns represent **objects (resources)**.

| Subject/Object | File A | File B | Printer |
| -------------- | ------ | ------ | ------- |
| User A         | R, W   | R      | X       |
| User B         | R      | W      | -       |
| User C         | -      | R, W   | X       |

Where:

* **R = Read**
* **W = Write**
* **X = Execute**
* **- = No Permission**

---

## Working of Access Control Matrix

Suppose **User A** wants to access **File A**.

* The system checks the matrix.
* Entry for User A and File A is **R, W**.
* Therefore, User A can read and write the file.

Similarly, if **User C** tries to access File A, the entry is "-", so access is denied.

---

## Advantages of Access Control Matrix

### 1. Simple and Easy to Understand

Clearly shows who has access to which resource.

### 2. Flexible

Different permissions can be assigned to different users.

### 3. Improves Security

Prevents unauthorized access to resources.

### 4. Centralized Access Management

All permissions are maintained in one structure.

---

## Limitations of Access Control Matrix

### 1. Large Size

In large organizations, the matrix becomes very large and difficult to manage.

### 2. Storage Overhead

Many entries may remain empty, wasting storage space.

### 3. Complex Maintenance

Updating permissions for thousands of users and resources can be difficult.

---

## Conclusion

The **Access Control Matrix** is a security model that specifies the permissions of subjects on objects. It helps control access to resources, prevents unauthorized activities, and improves overall system security. Although it may become large in complex systems, it remains a fundamental concept in information security and access control.
